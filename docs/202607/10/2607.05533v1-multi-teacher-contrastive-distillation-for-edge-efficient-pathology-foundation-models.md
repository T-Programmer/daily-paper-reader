## 摘要
计算病理学基础模型（PFMs）推动了全切片图像分析的发展。然而，其规模和推理成本阻碍了在病理科室的本地部署。我们提出MuCoDi，一种预训练框架，将来自多个PFMs的冻结切片嵌入蒸馏到紧凑的面向边缘编码器中。MuCoDi不是回归单个教师特征，而是采用从MoCo v3改编的对比蒸馏目标，训练轻量级MobileOne和RepViT学生模型，其中缓存的Virchow2、UNI2和H-Optimus-1嵌入替代了动量编码器键。我们在仅11.8K张WSI的14.3M TCGA切片上预训练学生模型，并在23个临床整理的下游分类任务上评估冻结编码器。基于RepViT的MuCoEdge学生模型在保持接近教师性能的同时将模型大小降低了几个数量级：MuCoEdge-R2.3和MuCoEdge-R1.5达到71.0%的外部AUROC，比最佳教师（Virchow2，71.8%）仅低0.8个百分点，而MuCoEdge-R2.3获得最佳外部F1和次佳AUPRC（51.8%和53.3%）。MuCoEdge-R1.0在仅6.4M参数和1.12 GFLOPs下达到70.9% AUROC。在树莓派5上，亚百万参数的MobileOne学生模型在保留66.5%至66.9%外部AUROC的同时，单切片推理速度比Virchow2提升高达605倍，证明具有PFM质量的病理表征可以走向实际边缘部署。代码见https://anonymous.4open.science/r/mucodi-6243。

## Abstract
Computational pathology foundation models (PFMs) have advanced whole-slide image analysis. However, their size and inference cost hinder local deployment in pathology departments. We propose MuCoDi, a pretraining framework that distills frozen tile embeddings from multiple PFMs into compact edge-oriented encoders. Instead of regressing individual teacher features, MuCoDi trains lightweight MobileOne and RepViT students with a contrastive distillation objective adapted from MoCo v3, where cached Virchow2, UNI2, and H-Optimus-1 embeddings replace momentum-encoder keys. We pretrain students on 14.3M TCGA tiles from only 11.8K WSIs and evaluate frozen encoders on 23 clinically curated downstream classification tasks. RepViT-based MuCoEdge students retain near-teacher performance while reducing model size by orders of magnitude: MuCoEdge-R2.3 and MuCoEdge-R1.5 reach 71.0% external AUROC, within 0.8 percentage points of the best teacher (Virchow2, 71.8%), while MuCoEdge-R2.3 obtains the best external F1 and the second-best AUPRC (51.8% and 53.3%). MuCoEdge-R1.0 reaches 70.9% AUROC with only 6.4M parameters and 1.12 GFLOPs. On a Raspberry Pi 5, sub-million-parameter MobileOne students achieve up to 605-fold single-tile speedup over Virchow2 while retaining 66.5% to 66.9% external AUROC, demonstrating that PFM-quality pathology representations can be moved toward practical edge deployment. Code is available at https://anonymous.4open.science/r/mucodi-6243.

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程
- **核心思想**：提出 **MuCoDi（Multi-teacher Contrastive Distillation）**，使用 MoCo v3 风格的对比蒸馏框架，将多个冻结的 PFM 教师（Virchow2、UNI2、H-Optimus-1）的 tile 嵌入作为固定键（key），训练轻量学生编码器（MobileOne 和 RepViT），避免直接回归教师特征，从而保留教师之间的互补关系。
- **关键技术细节**：
  - 学生网络：MobileOne（极轻量、可重参数化卷积）和 RepViT（ViT 启发的高效 CNN）系列。
  - 教师嵌入预计算并缓存，冻结教师，仅优化学生骨干和每个教师对应的线性投影头（投影到 1280 或 1536 维）。
  - 采用 MoCo v3 的两视图数据增强（随机裁剪、颜色抖动、灰度化、水平翻转、高斯模糊、太阳化）。
  - 损失函数：多教师对比损失（InfoNCE），对每个教师的两个视图求和，并缩放梯度。
- **公式流程（文字说明）**：
  1. 对每个训练 tile x，采样两个增强视图 x₁, x₂。

  3. 预计算该 tile 的教师嵌入 k_t（作为正键），当前全局批次中其他 tile 的教师嵌入作为负键。
  4. 对每个教师和视图计算 InfoNCE 损失（式 1），最终损失为所有教师和视图的损失之和（式 2）。

### 6. 论文的主要结论与发现
- MuCoEdge 学生模型在保持接近教师性能的同时，参数量和计算量降低 1-3 个数量级：MuCoEdge-R2.3（22.4M 参数）外部 AUROC 71.0%，仅比最佳教师 Virchow2（71.8%）低 0.8pp；外部 F1 和 AUPRC 甚至最优。

- MuCoEdge-R0.9（4.7M 参数）以 18× 更少参数、21× 更少 GFLOPs 超越 LiteFM-L（86.6M 参数）的外部 AUROC（70.5% vs 69.9%）。
- 证明仅使用 11.8K WSI（比 LiteFM 少 6×，比 Virchow2 少 260×）即可蒸馏出高质量的小模型，实现边缘部署可行性。

### 7. 优点：方法或实验设计上的亮点
- **对比蒸馏策略**：避免直接教师特征回归，利用多教师间的对比关系，保留互补知识。
- **数据效率**：学生预训练仅使用 TCGA 的 11.8K WSI（较教师模型大幅减少），降低数据门槛。
- **架构多样性**：同时测试卷积（MobileOne）和混合（RepViT）学生，覆盖不同计算预算。
- **实际边缘部署评估**：在树莓派 5 上测量真实单 tile 延迟，提供有意义的部署参考。
- **公平基准**：所有模型采用统一 MIL 评估流程（STAMP），并报告置信区间，减少偶然性。

### 8. 不足与局限
- **实验覆盖局限**：
  - 仅使用 TCGA 单中心数据预训练，缺乏多中心、多染色、不同分辨率的泛化验证。
  - 下游任务仅 23 个二元分类，未能涵盖更复杂的多分类、分割、生存分析等。
  - 缺乏消融实验（如单教师 vs 多教师、不同投影头、温度调参等）。
- **偏差风险**：教师模型本身也是基于公开数据训练，可能包含种族、诊断偏差，蒸馏后可能被继承。
- **应用限制**：
  - 仅支持 2.0 μm/px 分辨率，其他分辨率未测试。
  - 未涉及量化、INT8 推理、模型剪枝等进一步压缩手段，推理延迟测量为 float32 CPU 模式。
  - 边缘部署仅测试树莓派 5，未覆盖更多常见工作站或边缘设备。
- **对比公平性**：LiteFM 在预训练中使用了包含 CPTAC 的数据，而 MuCoDi 仅用 TCGA，可能使外部验证结果对 LiteFM 不利。
